How it works

From URL to risk level in four steps.

01

Normalize and validate

The app trims the submitted URL, adds a scheme when needed, and rejects empty or malformed input before prediction.

02

Extract URL features

Feature extraction counts structural and lexical indicators such as dots, hyphens, digits, query parameters, IP addresses, and suspicious words.

03

Predict locally

A Random Forest classifier loaded from model.pkl returns safe and phishing probabilities for the feature vector.

04

Explain the result

The UI displays the prediction, confidence, traffic-light risk level, and feature table so the user can understand why the link looks risky.
